Практическая реализация технологии «с нуля»

Данное руководство описывает процесс создания одностраничного веб‑приложения на «голом» Python (без фреймворков), которое раздаёт персонализированные аффирмации. В основе лежит кастомный HTTP‑сервер, написанный с использованием встроенной библиотеки socket. Сверх базового HTTP‑сервера добавлена Машина аффирмаций.

📐 Архитектура решения

Проект состоит из следующих логических блоков:

  • Сокет‑сервер — принимает TCP‑соединения, читает сырой HTTP‑запрос.
  • Маршрутизатор (handle_request) — определяет, какой ресурс запрошен (/, /next-affirmation, статика).
  • Менеджер аффирмаций (AffirmationManager) — управляет очередью уникальных фраз, исключая повторения.
  • Генератор HTML (generate_html) — встраивает текущую аффирмацию в готовый шаблон.
  • Клиентский JavaScript — асинхронно обновляет контент без перезагрузки страницы.

Шаг 1: HTTP‑сервер на сокетах

import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(('localhost', 8080))
server.listen(5)
print("Сервер на http://localhost:8080")

while True:

Функция handle_request() парсит URI и возвращает соответствующий контент (HTML, JSON, SVG или 404).

Шаг 4: Логика аффирмаций без повторов

class AffirmationManager:
    def get_next(self):
        if not self.remaining:
            self.remaining = [a for a in self.original_list if a not in self.history]
            random.shuffle(self.remaining)
        next_aff = self.remaining.pop(0)
        self.history.append(next_aff)
        return next_aff

Шаг 5: AJAX-эндпоинт /next-affirmation

Возвращает JSON с новой аффирмацией и обновлённой статистикой. Клиент обновляет DOM без перезагрузки.

Шаг 6: Раздача статических SVG-файлов

Сервер читает файлы logo.svg и hand-help.svg из директории и отдаёт с правильным Content-Type.

Модификация: что добавлено сверх базового HTTP‑сервера

Сверх базового HTTP‑сервера добавлена Машина аффирмаций — компонент, который обеспечивает интеллектуальную выдачу уникальных аффирмаций, динамическое обновление интерфейса и подсчёт прогресса. Вот её ключевые детали:

  • Машина аффирмаций (AffirmationManager) – управляет очередью уникальных фраз, случайным образом перемешивает оставшиеся аффирмации после каждого цикла и хранит историю просмотренных. Это гарантирует, что ни одна фраза не повторится, пока не будут показаны все.
  • Динамическое обновление контента (AJAX) – страница не перезагружается, при обновлении аффирмации добавлена микроанимация.
  • Визуальный счётчик прогресса – показывает, сколько уникальных аффирмаций уже просмотрено из общего числа.
  • Раздача статических SVG-файлов – корректная обработка изображений с fallback на 404.
  • Адаптивная вёрстка и дизайн – градиенты, карточки, тени, медиазапросы под мобильные устройства.
  • Логирование работы сервера – вывод в консоль деталей каждого запроса и состояния менеджера аффирмаций.
